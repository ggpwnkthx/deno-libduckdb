## Project Overview

This is a Deno FFI binding for DuckDB - a low-level TypeScript wrapper around
the native DuckDB C library. It provides typesafe FFI bindings to access
DuckDB's C API directly from Deno.

## Commands

```bash
# Download DuckDB native library
deno task download

# Generate FFI bindings from duckdb.h header
deno task generate

# Run tests
deno task test

# Full build (downloads, generates, formats, lints, checks, and tests)
deno task build
```

## Architecture

The project uses libclang to parse DuckDB's C header (`libduckdb/duckdb.h`) and
generate TypeScript FFI bindings automatically.

**Key modules:**

- `src/ffi.ts` - Generated FFI struct descriptors and function symbols
  (regenerated by `tools/generate.ts`)
- `src/ffi_config.ts` - DuckDB-specific type mappings: handle types (opaque
  pointers), buffer types, and C type to FFI type mappings
- `src/library.ts` - Library loading with auto-download fallback, symbol
  management
- `src/types.ts` - TypeScript type definitions for DuckDB handles and the
  DuckDBType enum

**Build flow:**

1. `tools/download.ts` fetches the native DuckDB library to `libduckdb/`
2. `tools/generate.ts` uses libclang to parse `libduckdb/duckdb.h` and outputs
   FFI bindings to `src/ffi.ts`
3. The generated `src/ffi.ts` is then linted, checked, and tested

**Testing:**

- Tests use the native library directly via `Deno.dlopen`
- Database handles, connections, and results are passed as `Uint8Array` buffers
  (8 bytes for pointers)
- Pointer values are read/written using `DataView.getBigUint64` and
  `getBigInt64`
